Why Citation-Based Activation Works

The transformer architecture routes different input tokens through different computational pathways. Citations exploit this to steer output toward specific methodologies.

Conor Brennan & Gerard Quinn

April 2026

This paper in context. This paper explains why citation-based activation works at the transformer architecture level. For the methodology it supports, see The Grounded Vibe Methodology. For the experimental evidence, see Grounding Produces Measurably Better Output. For a case study showing grounding patterns visible in production code, see Building a Production Tool with GVM.

The Short Version

When you tell an LLM to “use Wiegers’ methodology for requirements elicitation,” does it do anything different from “write good requirements”? The transformer architecture guarantees that different tokens produce different computations. Whether those different computations produce usefully different output is an empirical question — but the mechanism is real, and preliminary experiments point in the right direction.

Five findings from the machine learning literature explain the mechanism:

  1. Prompt tokens shift attention weights — the attention equation routes computation based on input (Vaswani et al., 2017)
  2. LLMs already learn from context — citations use the same conditioning channel as few-shot learning (Brown et al., 2020)
  3. Different tokens activate different circuits — circuit analysis shows token-specific computational pathways (Elhage et al., 2021)
  4. Context density amplifies activation — induction heads match patterns proportional to what’s in the context window (Olsson et al., 2022)
  5. Widely-referenced experts likely activate more reliably — a plausible inference from how training data shapes learned patterns (Kaplan et al., 2020)

A preliminary experiment (60 market risk dashboards, 360 independent reviews) found a statistically significant dose-response: more grounding produced progressively better output when measured against the cited experts’ published standards (F = 19.96, p < 0.001). The experiment has limitations — same-model evaluation, single task type — but it is consistent with the mechanism described here.

The Sceptic’s Question

A reasonable person encountering citation-based activation for the first time will ask: “Is this just prompt engineering with extra steps? Does naming an expert actually change what the model does, or does it just feel like it does?”

This is the right question. The answer requires looking inside the model — not at the output, but at the mechanism that produces it. Five papers, published between 2017 and 2022, collectively describe that mechanism. A sixth piece of evidence — our own experiment — provides preliminary empirical support. Each section below walks through one contribution.

1. Prompt Tokens Shift Attention Weights — That’s the Mechanism

Vaswani et al.’s self-attention equation defines how every transformer processes input:

Attention(Q, K, V) = softmax(QKT / √dk)V
Vaswani et al., “Attention Is All You Need,” NIPS (2017). The paper that replaced recurrence with attention as the foundation of sequence modelling.

Queries (Q), keys (K), and values (V) are derived from the input tokens. The dot product QKT computes how much each token matters to every other token. The softmax normalises these into attention weights. Different tokens in the prompt produce different weights, which produce different output.

When a prompt contains “Wiegers” and “Software Requirements,” those tokens participate in the QKT computation at every layer. They shift the attention weights, which shifts which learned patterns contribute to the output. That different tokens produce different computations is a consequence of the architecture. Whether those computations steer output in a useful direction is the empirical question addressed later in this paper.

2. LLMs Already Learn from Context — Citations Use the Same Channel

Brown et al. showed that GPT-3 could perform tasks specified entirely through the prompt — translation, question answering, arithmetic — without any parameter updates. The model learned the task from examples in the context window.

Brown et al., “Language Models are Few-Shot Learners,” NeurIPS (2020). Established in-context learning as a fundamental LLM capability.

If a model can learn a new task from prompt context, it is reasonable to expect it can be steered toward a specific methodology by providing that methodology’s key concepts. Citation-based activation uses the same conditioning channel as few-shot learning — tokens in the context window that shape the output distribution — but the conditioning signal is expert names, work titles, and technique descriptions rather than input-output examples. These are different types of conditioning signals, and the degree to which name-based citations produce the same quality of steering as worked examples is not established by Brown et al. The mechanism is the same; the efficacy of this particular application of it is what needs testing.

Brown et al. also showed a consistent pattern: more specific context produces more accurate output. Few-shot outperformed zero-shot. Task-specific prompts outperformed generic ones. This is consistent with the observation that “Wiegers, Software Requirements 3rd ed., context-free questions” produces more focused output than “write good requirements” — though it does not directly predict it, because citation-based prompts are structurally different from few-shot examples.

3. Circuit Analysis Shows Token-Specific Computational Pathways

Consider two prompts: one cites “Wiegers” for requirements elicitation, the other cites “Copeland” for boundary value analysis. Both are software engineering tasks. Both produce text. The attention equation guarantees that different input tokens produce different attention patterns. But do those different patterns activate meaningfully different knowledge?

Elhage et al., “A Mathematical Framework for Transformer Circuits,” Anthropic (2021). A foundational work in mechanistic interpretability of transformers. Note: circuit-level findings are from small (1–2 layer) models. Extrapolation to production-scale LLMs is a reasonable hypothesis, not a confirmed result.

Elhage et al. provided a rigorous description of how transformer components compose into interpretable circuits in small models. They showed that attention heads perform specific, identifiable operations — copying tokens, matching patterns, routing information. Their circuit analysis suggests that different tokens activate different computational pathways, which is consistent with the hypothesis that different citations produce different activations. The caveat: these findings come from 1–2 layer transformers. Whether the same circuit-level specificity holds in production-scale models (100B+ parameters) is plausible but not confirmed.

Separately, Elhage et al. formalised superposition in a later paper (“Toy Models of Superposition,” 2022): the model encodes more features than it has dimensions, with features overlapping in parameter space. This offers a possible explanation for how citations activate specific knowledge despite the model having no discrete “Wiegers module” — the knowledge is distributed across parameters, and the citation tokens may help resolve the superposition toward relevant features. But superposition also means features interfere with each other, so citation tokens could activate unintended features alongside the intended ones. The mechanism cuts both ways.

4. Reference Files Give Induction Heads More to Match Against

Olsson et al. identified a specific circuit — the induction head — and presented evidence that it is a key mechanism for in-context learning. Induction heads look for sequences in the current context that match patterns seen earlier in the context, and use those matches to predict what comes next.

Olsson et al., “In-context Learning and Induction Heads,” Anthropic (2022). Presented evidence that induction heads are a key mechanism underlying in-context learning. The paper studied models up to ~13B parameters.

This is consistent with the observation that GVM reference files — verbose documents loaded into context before generation — produce stronger activation than bare citations. A reference file containing “Wiegers — context-free questions, requirements classification (business, user, functional), ambiguity indicators, MoSCoW prioritisation” gives the induction heads more terms to potentially match against. A bare citation (“use Wiegers”) gives them fewer.

The inference that more context terms means proportionally stronger activation is plausible but simplified. Induction heads match specific sequential patterns, not arbitrary keywords. Whether additional terminology in a reference file creates matchable patterns for a given generation task depends on the specific content and the task. The relationship is likely positive but not guaranteed to be linear.

5. Widely-Referenced Experts Likely Activate More Reliably

Kaplan et al., “Scaling Laws for Neural Language Models,” arXiv:2001.08361 (2020). Established power-law relationships between total training data, model size, and aggregate model performance. Note: the paper studied aggregate scaling, not per-topic performance.

Kaplan et al. established power-law relationships between training data volume, model size, and overall model capability. Their work studied aggregate performance (cross-entropy loss across all tasks), not per-topic performance. They did not measure whether a model performs better on topic X when it has seen more training data about topic X specifically.

However, GVM practitioners observe a pattern that is consistent with the intuition behind scaling laws: citing a widely-referenced expert like Wiegers (whose work appears across textbooks, courses, blog posts, and professional discussions) seems to produce more reliable activation than citing an emerging expert with a single niche publication. The plausible explanation is that more frequent exposure during training creates stronger, more robust learned patterns. But we do not know what is in any model’s training corpus, and Kaplan et al.’s results do not directly establish per-topic scaling. This section presents a hypothesis consistent with the scaling literature, not a finding derived from it.

GVM’s expert scoring system reflects this hypothesis. The “Authority” and “Adoption” dimensions measure how widely an expert’s work is referenced — an indirect proxy for likely training data frequency. A Canonical expert (score 4.5+) is, under this hypothesis, more likely to have robust representation in the model’s parameters than a Provisional one (below 1.5). The scoring system operationalises a reasonable bet, not a proven law.

6. Panels Constrain the Output Under Multiple Expert Influences Simultaneously

When GVM assembles a panel of 3–7 experts, each name creates a separate set of attention patterns via multi-head attention. The model has multiple attention heads per layer, and different heads can attend to different expert citations simultaneously.

The output at each generation step is conditioned on the joint presence of all cited experts:

P(yt | y<t, expert1, expert2, ..., expertn)

This is not an average of individual expert-conditioned distributions. It is a single distribution conditioned on all experts simultaneously. The softmax normalises across all tokens, including all expert citations, producing weights that reflect the joint influence of the entire panel.

The practical consequence: panel output is jointly constrained by all cited experts — text that scores high under the joint distribution, meaning it tends to be consistent with Wiegers’ structure, Patton’s workflow narratives, Cooper’s personas, and Christensen’s jobs-to-be-done simultaneously. This is more constrained (and therefore more specific) than any single expert’s conditioning alone. Note that joint conditioning can also produce text that no single expert would generate individually — it is a confluence of influences, not a strict intersection where only shared content survives.

7. Preliminary Evidence: The Dashboard Experiment

The mechanism sections above explain why citation-based activation should work. This section describes a preliminary experiment that tested whether it does.

Sixty market risk dashboards were generated under three grounding conditions:

  • Group 1 (20 dashboards) — ungrounded. Task prompt only.
  • Group 2 (20 dashboards) — design-grounded. Task plus Tufte and Few with specific principles.
  • Group 3 (20 dashboards) — fully grounded. Design experts plus domain experts (Jorion, Hull) and writing experts (Doumont, Zinsser).

Dashboards were shuffled into random order and stripped of group identifiers. All 60 were then blind-reviewed twice — once by ungrounded reviewers using their own criteria, once by expert-grounded reviewers using criteria derived from the cited experts’ published principles. No reviewer saw more than one dashboard, and no reviewer knew which grounding condition had produced the dashboard under review. The same dashboards, two different measurement instruments, 360 total reviews.

The result

Under ungrounded review, Group 2 scored below Group 1 (7.37 vs 7.59). Design-only grounding appeared to make things worse. Under expert-grounded review, the ranking corrected to a clean dose-response: G1 (6.83) < G2 (7.22) < G3 (7.65). The interaction was statistically significant (F = 19.96, p < 0.001).

The largest effects were on the dimensions most directly addressed by the grounding experts: chartjunk absence (Tufte, d = 3.67 for G1 vs G2), data-ink ratio (Tufte, d = 3.87), and message-driven structure (Doumont, d = 7.4 for G1 vs G3). Dimensions where the grounding was less specific (domain metrics from Jorion and Hull) showed no significant group differences.

What this experiment does and does not establish

It establishes: Under expert-grounded review criteria, grounded output scores significantly higher than ungrounded output, with a dose-response pattern. The improvement is concentrated on the dimensions directly addressed by the grounding experts.

It does not establish: The experiment used same-model evaluation (Claude generated the dashboards and reviewed them). The human confirmation of review criteria partially breaks this loop, but full validation requires human reviewers or cross-model replication. It covers one task type (market risk dashboards). The effect may differ for other document types. See Grounding Produces Measurably Better Output for detailed limitations and per-dimension results.

What Remains Open

The mechanism is described. Preliminary evidence is encouraging. Four questions remain:

  1. How large is the effect with human evaluators? The dashboard experiment used AI reviewers. Human expert reviewers may confirm, amplify, or contradict the findings. Until replicated with human evaluation, the results demonstrate that grounding changes what the model produces in ways that expert-informed evaluation recognises as improvement.
  2. What is the optimal panel? The attention mechanism explains why panels work, but not how to select the best one. How many experts? Which combinations? When do additional experts produce diminishing returns or conflicting signals?
  3. Does it transfer across models? Preliminary cross-model results are reported in Grounding Produces Measurably Better Output for Claude Opus 4.6, GPT-5.4, and Gemini 3.1 Pro — the dose-response pattern and ranking reversal appear in all three. The mechanism is architecture-dependent (Vaswani et al.’s attention equation applies to all transformers), so cross-architecture portability is theoretically expected. Same-model evaluation remains a confound (the same model generates and scores the output); cross-model review is acknowledged in the experiment paper as the largest remaining experimental gap.
  4. Does the effect depend on task type? The dashboard experiment tested a task that is primarily evaluative and structural — designing a layout, selecting visualisations, applying data presentation principles. This is an alignment-dependent task where the expert’s framework defines what “good” looks like. Code generation is a different class of task: pretraining-dependent, requiring precise recall of framework-specific APIs and method signatures. Observations from a production build (see The Grounded Vibe Methodology, Section 9.1) found that expert grounding did not prevent either framework-specific errors or principle violations during code construction, while the same experts grounding the review phase caught those issues immediately. The PRISM study (Hu, Rostami & Thomason, USC, March 2026) provides a possible mechanism: personas improve alignment-dependent tasks but damage accuracy on pretraining-dependent tasks. However, the production observation lacks a control group — we do not know whether an ungrounded build would have produced more or fewer issues. Whether citation-based activation is effective, inert, or counterproductive during code generation is an open empirical question that the current evidence does not resolve. The preliminary hypothesis is that expert grounding is effective for declarative tasks (evaluation, review, document structure, requirements, test design) and undemonstrated for procedural tasks (code construction, API usage). If confirmed, this would be a boundary condition on the mechanism, not a refutation of it — the attention-shifting effect is real regardless, but the downstream impact on output quality may depend on whether the task benefits from the shifted distribution.
The grounded position

The transformer architecture provides a mechanistic explanation for why citation-based activation should work. Brown et al.’s in-context learning results demonstrate the prerequisite capability. Elhage et al. and Olsson et al. describe the internal machinery, with the caveat that circuit-level findings come from smaller models. A preliminary experiment with 60 dashboards and 360 reviews found a statistically significant dose-response under expert-grounded evaluation. What remains is replication with human reviewers, testing across task types — particularly the open question of whether the effect differs between evaluative and construction tasks — cross-model portability, and identifying the boundary conditions.
