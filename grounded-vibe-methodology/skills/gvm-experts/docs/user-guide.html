/gvm-experts

Score, rescore, discover, and manage experts in the GVM roster.

Overview

/gvm-experts is the user-facing command for expert roster management. It provides six operations for working with the expert reference files that all GVM pipeline skills depend on.

The expert reference files (the roster) live in the plugin install path (~/.claude/skills/gvm-design-system/references/). User data — the activation log, discovered experts, and rescore history — lives separately in ~/.claude/gvm/, so it survives plugin updates. This skill is the interface for working with both.

Most pipeline skills score experts automatically when they encounter unscored entries (per shared rule 9). Use /gvm-experts when you want to explicitly review, rescore, or add experts outside of a pipeline phase.

Operations

When you run /gvm-experts, you are presented with six options:

Table 1 — Expert management operations
OperationWhat it does
Score experts in a reference fileSelect a file and score all unscored experts using independent verification
Rescore a specific expertRe-evaluate a single expert’s scores with fresh assessment
Rescore all experts in a fileRe-evaluate every expert in a selected file sequentially
Score all unscored expertsIterates through unscored experts across all reference files, or industry domain files only — you choose the scope at invocation
Add a new expertAdd an expert to a reference file using the Expert Discovery and Persistence Process
Review utilisation dataAnalyse the activation log — which experts are loaded vs cited, divergence signals, tier breakdown

When to Use

  • When you disagree with an expert’s classification — rescore them
  • When adding a domain you haven’t used before — add experts manually
  • Periodically — review utilisation data to check whether your roster matches actual usage

How Scoring Works

Every expert is scored on five dimensions (Authority, Publication, Breadth, Adoption, Currency) and every work on four (Specificity, Depth, Currency, Influence). Scores determine classification: Canonical (4.5+), Established (3.5–4.4), Recognised (2.5–3.4), Emerging (1.5–2.4), or Provisional (below 1.5).

Scoring uses independent verification: a second agent assesses the same expert without seeing the first agent’s scores. If the two assessments converge, the averaged scores are used. If they diverge, you decide.

Automatic scoring

Pipeline skills score unscored experts automatically when they load reference files (shared rule 9). You do not need to run /gvm-experts before every pipeline phase. Use it when you want to review scores, rescore after new information, or add experts manually.

Here is what it looks like to score a single expert using “Rescore a specific expert”:

GVM scores Fowler across 5 dimensions, then dispatches a second
agent for independent verification.

Agent 1 scores:

  → Average: 4.8 → Canonical

Agent 2 scores (independent):
  Authority: 5   Publication: 5   Breadth: 5   Adoption: 5   Currency: 4
  → Average: 4.8 → Canonical

Convergence: both agents agree. Averaged scores persisted.

Work: "Refactoring" (2nd ed.)
  Specificity: 4   Depth: 5   Currency: 4   Influence: 5
  → Average: 4.5 → Canonical

  Specificity: 4   Depth: 5   Currency: 3   Influence: 5

Scores written to architecture-specialists.md.
If the two agents diverge (e.g., one scores Currency as 3 and the other as 5), both assessments are presented and you decide which to use.

Utilisation Feedback

The activation log (~/.claude/gvm/expert-activations.csv) tracks which experts are loaded into context and which are actually cited in output. Over time, this data reveals whether classifications match actual usage:

  • >80% cite rate — upward signal. The expert is used more than their classification suggests.
  • <40% cite rate — downward signal. The expert is used less than expected.

The “Review utilisation data” operation surfaces these signals. Utilisation data is a signal, not a verdict. You decide whether to act on it.

User data lives in ~/.claude/gvm/ and is never overwritten by a plugin update. The plugin install path (~/.claude/skills/gvm-design-system/references/) contains the canonical roster files — these are updated when you rescore or add experts through this skill.

Pipeline Position

/gvm-experts sits outside the sequential pipeline. It is available at any time — before, during, or after a pipeline run. It manages the expert roster that other skills consume, but no pipeline phase depends on it being run explicitly. Expert scoring happens automatically via shared rule 9 whenever unscored experts are encountered.
