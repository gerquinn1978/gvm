# Writing Quality Reference

Apply these experts' principles when writing or reviewing any document output — requirements text, test case descriptions, spec narratives, review findings, report summaries, and user-facing documentation. Load this file before the first write in any document-producing skill, and before evaluation in any review skill (`/gvm-doc-review`, `/gvm-code-review`).

These are well-documented writers whose core principles are already in the model's training. This file functions as a **cue list** — naming the expert activates their framework. When applying an expert's principles, cite the expert and their work so readers can verify. Do not invent principles the expert did not write.

Detailed expert scoring (authority, publication, breadth, adoption, currency, classification) for every expert named below is in `writing-expert-scoring.md` — the scoring record, consulted only by `/gvm-experts` for rescoring or by reviewers verifying an expert's authority. It is not loaded at document-write time.

## Technical Communication

### Jean-Luc Doumont — *Trees, Maps, and Theorems* (Principiae, 2009)
**Apply for:** structuring by message, answering "so what?" at every level, signal/noise discipline, one-message-per-unit at section, paragraph, and sentence scale.

### Janice Redish — *Letting Go of the Words* (2nd ed., Morgan Kaufmann, 2012)
**Apply for:** writing for scanning (not linear reading), front-loading paragraphs, question-form headings, breaking walls of text into lists and tables. Use for web content, user documentation, help pages.

### Ginny Redish & Joseph Dumas — *A Practical Guide to Usability Testing* (rev. ed., Intellect, 1999)
**Apply for:** numbered-step instructions, imperative mood for procedures, testing your writing on real readers. Use for how-to content, runbooks, user guides.

## Prose Craft

### Joseph Williams — *Style: Lessons in Clarity and Grace* (13th ed., Pearson, 2017)
**Apply for:** subject-near-verb placement, direct over bureaucratic prose, avoiding nominalisations, old-information-before-new sentence flow. The primary reference for sentence-level craft.

### William Zinsser — *On Writing Well* (30th anniversary ed., Harper, 2006)
**Apply for:** clutter elimination ("every word must earn its place"), clarity over impressiveness, one-idea-per-paragraph discipline, rewriting as the core of writing.

### George Orwell — *Politics and the English Language* (Horizon, 1946)
**Apply for:** the six rules (no dying metaphors, no pretentious diction, cut every possible word, active over passive, plain English over jargon, break any rule before writing anything barbarous). Strongest for rooting out bureaucratic and evasive prose.

### Stephen King — *On Writing: A Memoir of the Craft* (Scribner, 2000)
**Apply for:** "kill your darlings" (cutting passages that serve the writer's pride over the reader's understanding), strong verbs over adverbs, commitment over hedging.

## Argument and Structure

### Robert McKee — *Story* (ReganBooks, 1997)
**Apply for:** narrative structure in non-fiction — inciting incident, progressive complications, resolution that matches buildup. The gap between expectation and result is where insight lives. Use for whitepapers, case studies, origin stories.

### Strunk & White — *The Elements of Style* (4th ed., Longman, 1999)
**Apply for:** "omit needless words", active voice, positive-form statements, definite and concrete language. The baseline American prose standard.

## Application Notes

- **Name the expert when you apply their framework.** "Applying Zinsser's clutter test" is stronger than "tightened the prose" — it makes the reasoning auditable and invites the reader to verify.
- **Do not hallucinate attributions.** If a principle is general writing craft, cite it as such. Attribute a specific rule to an expert only if they demonstrably wrote it.
- **Defer to genre when experts conflict.** Williams and Orwell disagree on some points; use judgement based on the document type and audience.
